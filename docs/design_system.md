# Supertime — Design System v1

## Philosophy

Supertime is not a productivity SaaS platform.
It is calm communication infrastructure for humans, creators, artists, thinkers and communities.

The interface should feel:

* calm
* breathable
* transparent
* emotionally warm
* intentional
* lightweight
* internet-native
* human-first

The design system should disappear into the background and allow:

* conversations
* people
* creator identities
* emotional interactions

…to become the focus.

---

# Core Design Principles

## 1. Invisible Interface

The UI should not scream for attention.
The interface acts as infrastructure.

Reduce:

* visual noise
* unnecessary borders
* heavy shadows
* gradients
* aggressive colors
* excessive UI chrome

---

## 2. Typography-led hierarchy

Hierarchy should come primarily from:

* spacing
* typography
* rhythm
* layout

Not from:

* colors
* giant cards
* decoration

---

## 3. Mobile-first always

All screens must be designed mobile-first.

Important:

* comfortable thumb reach
* large touch targets
* minimal cognitive load
* low visual density
* lightweight rendering

---

## 4. Calm interaction design

Avoid dopamine-driven UX.

No:

* aggressive notifications
* engagement traps
* endless feeds
* gamification overload
* attention-maximizing patterns

---

## 5. Creator individuality inside system consistency

The platform should feel coherent.

BUT:
creators should gradually be able to express individuality.

Think:
"custom rooms inside a shared architecture."

Not:
"every profile becomes a completely different website."

---

# Shadow System

Minimal shadows.

Prefer:

* subtle borders
* tonal separation
* spacing

---

# Core Components

## Buttons

### Primary Button

* black background
* white text
* medium weight
* rounded full or large radius

### Secondary Button

* transparent background
* subtle border

### Ghost Button

* text only
* low emphasis

Important:
Buttons should feel quiet and intentional.

---

## Cards

Cards should:

* feel lightweight
* avoid heavy outlines
* use spacing for structure
* breathe vertically

Avoid:

* dense dashboard cards
* enterprise SaaS styling

---

## Creator Cards

The creator card is the core object of the platform.

Every creator card should communicate:

* humanity
* personality
* emotional tone
* session intent
* warmth

Avoid:

* LinkedIn energy
* resume aesthetics
* over-optimization

Better:
"Late-night script practice"
than:
"Acting Consultant with 5 years experience"

---

# Navigation

Navigation should feel:

* lightweight
* calm
* invisible

Prefer:

* simple bottom tabs on mobile
* minimal labels
* low visual weight

---

# Motion

Motion should be subtle.

Avoid:

* flashy transitions
* overly animated interfaces
* attention-seeking motion

Use motion only to:

* reduce friction
* improve continuity
* create softness

---

But customization must remain inside the Supertime ecosystem.

Milkshake meets Are.na.

Structured freedom.
